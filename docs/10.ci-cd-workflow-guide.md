# CI/CD Workflow 설정 가이드 (Public GitHub + Free Cloud)

GitHub Actions + GitHub Container Registry(GHCR) + GitOps 기반으로,
퍼블릭 레포에서 무료로 시작할 수 있는 배포 흐름을 설명합니다.

---

## 전체 흐름

```text
코드 Push
  └─► [1. determine-environment]  브랜치 기준으로 dev/prod 환경 결정
        └─► [2. build]             Docker 빌드 → GHCR push
              └─► [3. deploy]      Helm values image tag 업데이트 → git commit/push (GitOps)
                    └─► (Optional) Deploy Hook 호출 (Vercel/Cloudflare/Render)
```

---


## Step 1. GitHub Public Repository 생성

1. [github.com](https://github.com) → 우측 상단 `+` → `New repository`
2. 저장소를 `Public`으로 생성
3. 로컬 코드 연결

```bash
git remote add origin https://github.com/skcc-wooyoungmoon/<repo-name>.git
git branch -M main
git push -u origin main
```

---

## Step 2. GitHub Actions 활성화 확인

- Repository 상단 `Actions` 탭 클릭
- 첫 실행 안내가 나오면 활성화 승인

---

## Step 3. GitHub Environments 생성

1. Repository → `Settings` → `Environments`
2. 아래 2개 환경 생성

| 환경명 | 용도 |
|--------|------|
| `dev-env` | `develop`, `feature/*`, `hotfix/*` |
| `prd-env` | `main` |

3. `prd-env`에는 필요 시 `Required reviewers` 설정

---

## Step 4. GHCR Push 권한 설정

1. Repository → `Settings` → `Actions` → `General`
2. `Workflow permissions` 섹션에서 `Read and write permissions` 선택 → `Save`

**왜 필요한가:**  
이 설정을 하면 GitHub Actions에서 자동으로 생성되는 `GITHUB_TOKEN` 토큰이 `secrets.GITHUB_TOKEN`으로 워크플로우에서 사용 가능해집니다.  
워크플로우의 `docker/login-action@v3` 스텝에서 이 토큰을 이용해 GHCR에 이미지를 push할 수 있습니다.

**워크플로우 코드 예시 (자동 설정됨):**
```yaml
permissions:
  contents: read
  packages: write  # 이 권한이 필요
```

---

## Step 5. 퍼블릭 무료 클라우드 연결 (옵션)

프로젝트 성격별 권장:

- Frontend(Next.js): `Vercel`
- Frontend(범용 정적/엣지): `Cloudflare Pages`
- Backend API: `Render`
- 컨테이너 직접 배포: `Fly.io`

### 5-1) Vercel / Cloudflare / Render Deploy Hook

각 플랫폼에서 Deploy Hook URL을 발급해 GitHub Secret에 저장합니다.

Repository → `Settings` → `Secrets and variables` → `Actions` → `New repository secret`

| Secret 이름 | 설명 |
|-------------|------|
| `VERCEL_DEPLOY_HOOK_URL_DEV` | dev 프론트 배포 훅 |
| `VERCEL_DEPLOY_HOOK_URL_PRD` | prod 프론트 배포 훅 |
| `CF_PAGES_DEPLOY_HOOK_URL_DEV` | dev Cloudflare Pages 훅 |
| `CF_PAGES_DEPLOY_HOOK_URL_PRD` | prod Cloudflare Pages 훅 |
| `RENDER_DEPLOY_HOOK_URL_DEV` | dev 백엔드 배포 훅 |
| `RENDER_DEPLOY_HOOK_URL_PRD` | prod 백엔드 배포 훅 |

워크플로우 하단의 주석 예시를 해제하면 `curl -X POST`로 배포를 트리거할 수 있습니다.

### 5-2) Fly.io (컨테이너)

- `flyctl auth token`으로 토큰 생성
- GitHub Secret에 `FLY_API_TOKEN` 추가
- `fly.toml` 준비 후 워크플로우 주석 해제

---

## Step 6. 워크플로우 TODO 값 수정

### Frontend 필수

```yaml
DEV_NEXT_PUBLIC_API_BASE_URL: https://api-dev.<your-domain>
PRD_NEXT_PUBLIC_API_BASE_URL: https://api.<your-domain>
```

### 공통

- 워크플로우의 optional deploy 블록에서 사용할 secret 이름 확인
- `fly.toml`/Deploy Hook URL 등 실제 값으로 교체

---

## Step 7. 트리거 동작 확인

| 브랜치 | 동작 |
|--------|------|
| `feature/*`, `hotfix/*`, `develop` push | `dev-env` 빌드/배포 |
| `main` push | `prd-env` 빌드/배포 |
| `workflow_dispatch` | 수동으로 `dev/prod` 선택 |

---

## 이미지 태그 규칙

```text
ghcr.io/<owner>/<app-name>:dev-20260401.42-a1b2c3d
```

형식: `<env>-<yyyymmdd>.<run-number>-<short-sha>`

---

## 자주 발생하는 오류

| 오류 | 원인 | 해결 |
|------|------|------|
| `denied: permission_denied` (GHCR push) | Actions 권한 read-only | Step 4 설정 확인 |
| `Environment 'prd-env' not found` | Environment 미생성 | Step 3에서 생성 |
| `No changes to commit` | tag 값 동일 | 정상 케이스 |
| `Push failed, exit 1` | 동시 push 충돌 | 워크플로우 재실행 |
| Deploy Hook 401/404 | Hook URL 오입력 | Secret 값 재확인 |
